# Stylelint

## Status
Accepted - 6/20/2026

## Reasoning
Stylelint will help me enforce good habits when it comes to my CSS. It should in theory catch any logical issues, disallow unpreferrable conventions, enforce patterns and limits, and specify certain notation for things like color selection. I'm still quite new to the world of frontend development, so I want to have something in place to catch me in case I fall. 

## Use Case
Stylelint will remain for the forseeable future throughout this entire process of making this webpage. I will require lots of CSS work to make the end result look appealing, so it will be in the background keeping my work in check while I run off to explore different options and potentially use AI generated code to help me produce quickly.

## Pros
1. Stylelint will keep my code in check, making sure that I don't fall into traps like hardcoding pixel positions.
2. Stylelint will allow me to comfortably use AI generated code without worrying about missing something logically or stylistic about the code.
3. Because I am using the default settings of stylelint, I'm hoping that it will teach me something about what good CSS looks like. 
## Cons
1. Stylelint will invalidate my styles in favor of its own. While this may make my code "better", it will also remove the personality from my code. Good or bad, this will result in the loss of my historical improvement, which would otherwise be fun to look at in the future.
2. Stylelint might make me complacent in any kind of AI generated code that I am implementing. This is something I need to look out for and make sure that I am reviewing everything being inputted consistently.
